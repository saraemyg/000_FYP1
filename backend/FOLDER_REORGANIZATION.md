## Key Points About This Layout

**Why `db/` is separate from `app/`?**
Both the FastAPI API (`app/`) and the ETL pipeline (`pipeline/`) need to access 
the database. By putting `db/` at the backend root, both can import from it:
- `from db.models import Detection`  (works in app/ AND pipeline/)

**Why TWO main.py files?**
- `backend/app/main.py` = FastAPI server (serves the web API for your React frontend)
- `backend/main.py` = Pipeline runner (processes videos, runs AI models)

They are completely separate processes. You run them independently:
```bash
# Terminal 1: Start the API server
uvicorn app.main:app --reload --port 8000

# Terminal 2: Run the ETL pipeline  
python main.py --camera cam_01
```

**What about `alembic/env.py`?**
Update the `target_metadata` line to point to your new models:
```python
# In alembic/env.py, change:
# target_metadata = SomeOldModel.metadata
# To:
from db.models import Base
target_metadata = Base.metadata
```

Then generate a migration:
```bash
cd backend/
alembic revision --autogenerate -m "add surveillance tables"
alembic upgrade head
```

## Docker Setup

Your `docker/` folder likely has something for FastAPI + PostgreSQL already.
Here's what the docker-compose should look like with the pipeline added:

```yaml
# docker/docker-compose.yml
version: "3.8"

services:
  # ── PostgreSQL Database ──
  db:
    image: postgres:16
    environment:
      POSTGRES_DB: surveillance_db
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data

  # ── Redis (cache + pub/sub for alerts) ──
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  # ── FastAPI Backend (serves API to React frontend) ──
  api:
    build:
      context: ../backend
      dockerfile: ../docker/Dockerfile.api
    ports:
      - "8000:8000"
    environment:
      DATABASE_URL: postgresql://postgres:postgres@db:5432/surveillance_db
      REDIS_URL: redis://redis:6379/0
    depends_on:
      - db
      - redis
    volumes:
      - ../backend:/app

  # ── ETL Pipeline (processes videos) ──
  # Only run when you need to process videos, not always-on
  pipeline:
    build:
      context: ../backend
      dockerfile: ../docker/Dockerfile.pipeline
    environment:
      DATABASE_URL: postgresql://postgres:postgres@db:5432/surveillance_db
      REDIS_URL: redis://redis:6379/0
    depends_on:
      - db
      - redis
    volumes:
      - ../backend/data:/app/data
      - ../backend/masks:/app/masks
    deploy:
      resources:
        reservations:
          devices:
            - capabilities: [gpu]   # GPU access for YOLO inference

volumes:
  pgdata:
```

## Step-by-Step: How to Merge Files Into Your Repo

```bash
# 1. Clone your repo
git clone https://github.com/saraemyg/000_FYP1.git
cd 000_FYP1/backend

# 2. Create new folders
mkdir -p db pipeline models masks data notebooks tests

# 3. Copy the files I gave you:
#    - db/models.py, db/database.py, db/schema.sql, db/seed.py, db/__init__.py
#    - pipeline/extractor.py, roi_pruner.py, detector.py, classifier.py,

#    - models/model_loader.py, models/__init__.py
#    - config.py, camera_config.json, main.py

# 4. Update requirements.txt (merge new deps with existing)
# 5. Update alembic/env.py target_metadata
# 6. Update .gitignore

# 7. Test it (see test script below)
```
